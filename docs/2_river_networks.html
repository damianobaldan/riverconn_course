River systems as networks

  • River systems can be represented as graphs/networks
  • Nodes (vertices) represent river segments
  • Links (edges) represent the connections (confluences and barriers) between segments
  • Many connectivity indices are defined for graphs, straightforward application to river systems when represented as graphs
  • Inclusion of barriers complicates the representation, but several indices can be adapted to account for them (e.g. DCI, Dendritic Connectivity Index)

The topology of river networks

  • Planar networks: no links (edges) intersections
  • Tree-like networks: no loops, only one path between any two nodes (vertices)
  • Number of nodes (vertices) and links (edges) in a tree-like network: E = V - 1
  • Weighted networks: node weights can represent attributes of river segments (e.g. length, discharge, etc.), edge weights can represent the strength of the connection (e.g. presence/absence of a barrier, etc.)

The igraph package

The igraph package

  • The igraph package implements routines for graphs and network analysis.
  • It can handle large graphs very well and provides functions for generating random and regular graphs, graph visualization, centrality methods and much more.
  • The packages allows for easy construction of igraph objects based on symbolic description of edges and vertices, edges and vertices lists, or adjacency matrices.
  • The book Statistical Analysis of Network Data with R by Kolaczyk and Csardi (2014) offers a comprehensive tutorial on the possibilities offered by the igraph package, including network data manipulation and visualization, descriptive analysis of network graphs, mathematical and statistical modeling of graphs, and network topology inference.

Centrality metrics with the igraph package

Centrality metrics with the igraph package

  • Centrality metrics are used to identify the most important vertices in a graph
  • Identify nodes that are relevant for connecting separate sections of the graph
  • Degree centrality: number of edges connected to a vertex (for weighted networks: strenght)
  • Betweenness centrality: number of shortest paths that pass through a vertex
  • Closeness (harmonic) centrality: inverse of the average shortest path length from a vertex
  • For directed and undirected graphs

Centrality metrics with the igraph package

Degree centrality of node \(i\) (\(k_{i}\)): number of edges connected to a vertex (for weighted networks: strenght)

\[k_i = \sum_{j}a_{ij}\] where \(a_{ij}\) is the element of the adjacency matrix that represents the presence (1) or absence (0) of an edge between vertices \(i\) and \(j\)

Centrality metrics with the igraph package

Betweenness centrality: number of shortest paths that pass through a vertex

\[g_i = \sum_{s \neq i \neq t} \frac{\sigma_{st}(i)}{\sigma_{st}} \] where where \(\sigma_{st}\) is the total number of shortest paths from node \(s\) to node \(t\) and \(\sigma_{st}(i)\) is the number of those paths that pass through node \(i\) (not where \(i\) is an end point)

Centrality metrics with the igraph package

Closeness centrality: inverse of the average shortest path length from a vertex

\[c_i = \frac{1}{\sum_{j \neq i} {d(ij)}} \] Harmonic centrality: for disconnected graphs (to avoid issues with \(d(ij) = \infty\))

\[c_i = \sum_{j \neq i} \frac{1}{ {d(ij)}} \]

where \(d(ij)\) is the shortest distance between nodes \(i\) and \(j\)

Centrality metrics with the igraph package

We create a graph, calculate degree, betweeness and closeness centrality and store the values as nodes attributes

g <- igraph::graph_from_literal( 1-+2, 2-+5, 3-+4, 4-+5, 6-+7,
  7-+10, 8-+9, 9-+10, 5-+11, 11-+12, 10-+13, 13-+12, 12-+14, 14-+15, 15-+16)
V(g)$degree <- igraph::degree(g)
V(g)$betweenness <- igraph::betweenness(g)
V(g)$closeness <- igraph::closeness(g, mode = "all")

Hydrographic data processing

The first step before building the graph is to identify confluences. To do this, the river network is split into the smallest segments allowed by its geometry.

shape_river_simple <- shape_river_small %>%
  st_as_sf %>%
  st_union()

river_to_points <- shape_river_simple  %>%
  st_as_sf() %>%
  st_cast("POINT") %>%
  mutate(id = 1:nrow(.))

Hydrographic data processing

Confluences are the points where three or more segments overlap.

joins_selection <- river_to_points %>%
  st_equals() %>%
  Filter(function(x){length(x) > 2}, .) %>%
  lapply(., FUN = min) %>%
  unlist() %>%
  unique()

river_joins <- river_to_points %>% 
  filter(id %in% joins_selection)

Integrating hydrographic and dams data

Integrating hydrographic and dams data

The position of confluences and dams are integrated in the same data.frame, that will be used to build the graph. The type of link (confluence or dam) is stored in the column type, while the columns pass_u and pass_d store the passability values for upstream and downstream movement, respectively.

network_links <- rbind(
  dams_snapped_reduced_joined %>% 
    mutate(type = "dam", id_barrier = id_dam) %>%
    dplyr::select(type, id_barrier, pass_u, pass_d),
  river_joins %>% mutate(type = "joint") %>%
    dplyr::select(type) %>%
    mutate(id_barrier = NA, pass_u = NA, pass_d = NA) %>%
    rename(geometry = x)) %>%
  mutate(id_links = 1:nrow(.))
